# ROADMAP.md

本文件描述 Request Bridge 后续的建议开发路线。

目标不是把所有功能一口气做满，而是按优先级逐步推进，让它越来越像一个可交付的桌面工具。

---

# Phase 1：可用化（当前阶段）

目标：
让工具从“能跑”走到“能操作”。

## 重点任务
- [ ] 首页工具台化布局
- [ ] 启动按钮和状态提示强化
- [ ] 请求记录列表 / 表格
- [ ] 配置编辑体验完善
- [ ] 规则增删改完善
- [ ] 保存配置后的反馈更清晰

## 完成标准
- 普通人打开后知道先点哪里
- 能在界面里完成主要配置
- 不需要手工改大段配置文件

---

# Phase 2：可交付化

目标：
让它更接近“可以发给别人直接用”。

## 重点任务
- [ ] 日志落盘
- [ ] 日志导出
- [ ] 错误弹窗 / 错误提示统一
- [ ] 首次使用引导
- [ ] 默认配置模板
- [ ] 更完善的 EXE 导出说明

## 完成标准
- 出问题时用户知道怎么排查
- 能把日志给别人
- 第一次使用不容易卡住

---

# Phase 3：Windows 成品化

目标：
做成更像正式软件的 Windows 交付物。

## 重点任务
- [ ] 便携版 EXE 测试
- [ ] 安装版 EXE 测试
- [ ] 应用图标
- [ ] 应用名称和版本信息
- [ ] 安装器体验优化
- [ ] 打包产物说明文档完善

## 完成标准
- EXE 可稳定运行
- 可以发给 Windows 用户直接使用
- 安装包与便携版都能分发

---

# Phase 4：长期使用能力

目标：
让它适合日常长期运行。

## 重点任务
- [ ] 系统托盘
- [ ] 开机自启动
- [ ] 自动更新
- [ ] 配置备份与恢复
- [ ] 请求记录筛选 / 搜索
- [ ] 更完整的诊断报告导出

---

# Phase 5：增强能力（可选）

不是当前最重要，但后面可考虑：

- [ ] SOCKS5 支持
- [ ] 更复杂的规则匹配
- [ ] 响应改写
- [ ] 请求录制
- [ ] 认证鉴权
- [ ] 多配置档案

---

# 当前最重要的一句话

不要把精力先花在“界面美化”上。

先把这几个做扎实：
1. 状态一眼看懂
2. 启动一键完成
3. 配置容易保存
4. 规则容易编辑
5. 日志容易查看
6. EXE 容易导出和分发
