# EXE_EXPORT.md

本文件专门写给：

- 不懂代码的人
- 需要把项目导出成 Windows EXE 的人
- 后续接手的人或 AI 助手

目标：
**让人能按步骤导出 EXE，并把 EXE 发到 Windows 电脑上直接使用。**

---

# 一、你最终要拿到什么

导出完成后，你主要关心的是 `dist/` 目录里的文件。

重点看这两种：

## 1. 便携版 EXE
特点：
- 不用安装
- 直接双击运行
- 最适合“先发给 Windows 电脑试用”

## 2. 安装版 EXE
特点：
- 像普通 Windows 软件一样安装
- 更适合正式分发

如果你不是技术人员，建议优先使用：

**便携版 EXE**

---

# 二、导出 EXE 前要准备什么

在当前项目目录里操作：

```bash
cd /home/node/clawd/project/request-bridge
```

然后安装依赖：

```bash
npm install
```

---

# 三、怎么导出 Windows EXE

执行：

```bash
npm run pack:win
```

导出完成后，去看：

```text
dist/
```

---

# 四、导出成功后怎么看文件

一般会在 `dist/` 里看到 Windows 打包产物。

你后续优先找：

- 便携版 `.exe`
- 或安装版 `.exe`

如果你只是想发到自己的 Windows 电脑上直接试，优先发：

**便携版 EXE**

---

# 五、拿到 EXE 之后怎么用

## 你的操作步骤

### 第 1 步：把生成好的 EXE 发到 Windows 电脑
可以通过：
- QQ
- 微信文件传输
- 网盘
- U 盘
- 局域网传输

### 第 2 步：在 Windows 电脑上保存到一个固定目录
建议不要放在系统临时目录。

例如可以放到：

```text
D:\RequestBridge\
```

### 第 3 步：双击 EXE
双击后应该打开桌面控制窗口。

### 第 4 步：在窗口里配置规则
按界面填：
- 监听端口
- 允许主机
- A 到 B 的映射规则

### 第 5 步：点击“启动代理”
启动后，软件开始监听。

### 第 6 步：在你的客户端或浏览器里设置代理
例如：
- 代理地址：Windows 电脑 IP
- 代理端口：你在软件里设置的端口

# 六、目标状态：开箱即用

这个项目当前正在往这个目标推进：

- 不要求最终用户装 Node
- 不要求最终用户改源码
- 不要求最终用户敲命令
- 尽量让最终用户只用 EXE 和图形界面

也就是说：

**最终交付物应该是软件，而不是开发项目。**

---

# 七、如果导出失败怎么办

先检查这几项：

## 1. 依赖是否安装完成
```bash
npm install
```

## 2. 语法是否正常
```bash
npm run check
```

## 3. 当前环境是否支持打包 Windows
有些 Linux 环境可以配好后打 Windows 包，但不是所有环境都天然可用。

如果当前环境打包失败，不代表项目坏了，可能只是当前机器的打包环境不完整。
