# 四、明确的产品方向

请不要把这个项目往“只给开发者用的脚本工具”方向带回去。

当前明确方向是：

**面向不懂代码的用户，做一个可视化、可打包 EXE、可直接双击运行的实用工具。**

所以在做决策时，请优先：
- 可理解
- 可操作
- 少命令行
- 少手改配置
- 错误提示清楚

而不是优先：
- 炫技
- 复杂架构
- 过度抽象

---

# 五、界面方向要求

用户已经明确给过参考图。

应参考的不是内容照抄，而是这些原则：

- 首页要像控制台
- 启动按钮必须显眼
- 当前状态必须一眼能看懂
- 请求记录应该成为主体区域
- 顶部要有映射目标和关键状态
- 排版偏工具型，不要过度装饰

# 七、协作要求（给 AI 和人都适用）

1. 重要改动必须同步更新文档
2. 不要只改代码不写文档
3. 不要把核心逻辑塞进前端
4. 配置项变动时必须同步更新：
   - `.env.example`
   - `src/config.js`
   - `docs/OPERATIONS.md`
5. 目标用户是普通用户，不是开发者

---

# 八、常见误区

## 误区 1：只要能跑就行
不行。这个项目目标是“能被普通人用”，不是“开发者自己能跑”。

## 误区 2：界面先随便，后面再说
可以先不美观，但不能不好用。功能入口必须清晰。

## 误区 3：让用户自己去改 JSON
尽量避免。应逐步把 JSON 隐藏在底层存储层。

---
